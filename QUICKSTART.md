# 🚀 Швидкий старт

Цей посібник допоможе вам швидко почати працювати з проектом.

## ⚡ За 5 хвилин

### Крок 1: Клонування та налаштування (1 хв)

```bash
cd /home/maestro/devel/ansible-mikrotik

# Автоматичне налаштування
./scripts/setup.sh
```

### Крок 2: Налаштування підключення (2 хв)

Відредагуйте `inventory/hosts.yml`:

```bash
nano inventory/hosts.yml
```

Змініть тільки пароль:

```yaml
wireguard_servers:
  hosts:
    srv-01.mood.pp.ua:
      ansible_host: "srv-01.mood.pp.ua"
      ansible_user: "admin"
      ansible_password: "ВАШ_ПАРОЛЬ_ТУТ"  # <-- Змініть це
```

### Крок 3: Перевірка з'єднання (30 сек)

```bash
source venv/bin/activate
make test
```

Якщо побачили `pong` - все готово! ✅

### Крок 4: Додавання першого клієнта (30 сек)

```bash
# Просто запустіть - все автоматично!
make add-client

# Система автоматично:
# 1. Знайде перший вільний IP (наприклад 10.66.66.5)
# 2. Створить ім'я: R65
# 3. Обчислить VNI: 65
# 4. Згенерує криптографічні ключі
# 5. Додасть peer на сервер
# 6. Створить конфігурацію з SSH ключем
```

**Або вказати конкретний IP:**
```bash
make add-client IP=10.66.66.12
# Створить R612 з VNI 612
```

### Крок 5: Імпорт на MikroTik клієнт

Конфігурація клієнта буде в `generated_configs/R65.rsc` (або інше ім'я)

**Що включено в конфігурацію:**
- ✅ WireGuard VPN з'єднання
- ✅ VXLAN для L2 мережі
- ✅ SSH ключ (MainSSH) вже доданий
- ✅ Firewall правила (захист від зовнішнього втручання)
- ✅ SSH доступ тільки по ключам з VPN
- ✅ Winbox тільки з VPN
- ✅ ether1 = WAN, інші ether = bridge
- ✅ WiFi інтерфейси вимкнені
- ✅ Захист від брутфорсу та сканування
- ✅ Клієнти bridge без доступу до інтернету

**Імпорт на роутер:**

1. Скопіюйте файл на MikroTik (через Winbox File Manager або SCP)
2. Підключіться через MAC-Winbox або консоль
3. Виконайте:

```
/import file=R65.rsc
```

**Після імпорту:**
- SSH: `ssh admin@10.66.66.5` (тільки з VPN мережі)
- Winbox: `10.66.66.5:8291` (тільки з VPN мережі)
- Пароль SSH вимкнено, тільки ключі!

## 📱 Основні команди

```bash
# Активувати віртуальне середовище
source venv/bin/activate

# Додати клієнта (конфігурація з SSH ключем вже готова)
make add-client NAME=peer5 IP=10.66.66.6

# Видалити клієнта
make remove-client NAME=peer5 IP=10.66.66.6

# Показати конфігурацію сервера
make show-config

# Перевірити з'єднання
make test

# Показати всі команди
make help
```

## ⚠️ Важливо

1. **Завжди активуйте venv перед роботою:**
   ```bash
   source venv/bin/activate
   ```

2. **SSH ключ MainSSH додається автоматично** до всіх клієнтів

3. **Ключі зберігаються локально** в `client_keys/` - не втрачайте їх!

3. **IP адреси повинні бути унікальними** - не використовуйте вже зайняті (10.66.66.1, .2, .4)

## 🆘 Проблеми?

### Не можу підключитися до сервера

```bash
# Перевірте з'єднання
ping srv-01.mood.pp.ua

# Перевірте чи правильний пароль в inventory/hosts.yml
```

### Помилка "collection not found"

```bash
ansible-galaxy collection install -r requirements.yml --force
```

### Помилка з Python бібліотеками

```bash
source venv/bin/activate
pip install -r requirements.txt
```

## 🎯 Типові сценарії

### Додати 5 клієнтів швидко

Створіть `clients.csv`:

```csv
client_name,client_ip
peer4,10.66.66.5
peer5,10.66.66.6
peer6,10.66.66.7
peer7,10.66.66.8
peer8,10.66.66.9
```

Виконайте:

```bash
make bulk-add FILE=clients.csv
```

### Тільки згенерувати ключі

```bash
make generate-keys NAME=peer10
```

### Перевірити що на сервері

```bash

```

Це покаже всі WireGuard peers, VXLAN інтерфейси та іншу конфігурацію.
