# 🏗️ Архітектура проекту

## Загальний огляд

Цей проект реалізує централізоване управління WireGuard VPN мережею з підтримкою VXLAN для створення розподіленої L2 мережі між MikroTik роутерами.

## 🌐 Мережева архітектура

### WireGuard мережа
- **Мережа**: 10.66.66.0/24
- **Сервер**: 10.66.66.1
- **Клієнти**: 10.66.66.2-254
- **Протокол**: UDP
- **Порт сервера**: 62356

### VXLAN архітектура

#### Версія 1.1.0+ (поточна)

**Концепція**: Окремий VXLAN інтерфейс для кожного клієнта

```
┌─────────────────────────────────────────────────────────┐
│                   MikroTik Сервер                        │
│                  (srv-01.mood.pp.ua)                     │
│                                                           │
│  ┌───────────────────────────────────────────────┐      │
│  │              Bridge (br_x)                     │      │
│  │           172.16.0.0/16 мережа                 │      │
│  │                                                 │      │
│  │  ┌──────────┐  ┌──────────┐  ┌──────────┐     │      │
│  │  │ vxlan-   │  │ vxlan-   │  │ vxlan-   │     │      │
│  │  │ peer1    │  │ peer2    │  │ peer3    │     │      │
│  │  └────┬─────┘  └────┬─────┘  └────┬─────┘     │      │
│  └───────┼─────────────┼─────────────┼───────────┘      │
│          │             │             │                    │
│          │             │             │                    │
│  ┌───────▼─────┬───────▼─────┬───────▼─────┐            │
│  │ WireGuard   │ WireGuard   │ WireGuard   │            │
│  │ Peer 1      │ Peer 2      │ Peer 3      │            │
│  │ 10.66.66.2  │ 10.66.66.3  │ 10.66.66.4  │            │
│  └─────────────┴─────────────┴─────────────┘            │
└─────────────────────────────────────────────────────────┘
              │             │             │
              │  WireGuard  │  WireGuard  │  WireGuard
              │   Tunnel    │   Tunnel    │   Tunnel
              ▼             ▼             ▼
        ┌──────────┐  ┌──────────┐  ┌──────────┐
        │ Client 1 │  │ Client 2 │  │ Client 3 │
        │  (peer1) │  │  (peer2) │  │  (peer3) │
        └──────────┘  └──────────┘  └──────────┘
```

**Переваги цього підходу:**
- ✅ Кожен клієнт має свій ізольований VXLAN інтерфейс
- ✅ Простіше управління (додавання/видалення інтерфейсу)
- ✅ Легше відстежити та налагодити з'єднання
- ✅ Можливість налаштовувати параметри окремо для кожного клієнта
- ✅ При видаленні клієнта видаляється весь інтерфейс (bridge port автоматично)

**Конфігурація на сервері для кожного клієнта:**
```routeros
# WireGuard peer
/interface wireguard peers
add name=peer1 interface=fly public-key="..." allowed-address=10.66.66.2/32

# Окремий VXLAN інтерфейс
/interface vxlan
add name=vxlan-peer1 vni=100 local-address=10.66.66.1 remote-address=10.66.66.2

# Додавання до bridge
/interface bridge port
add interface=vxlan-peer1 bridge=br_x
```

#### Версія 1.0.0 (застаріла)

**Концепція**: Один VXLAN інтерфейс з множинними VTEP

```
Сервер: 1 VXLAN інтерфейс → N VTEP endpoints
```

**Недоліки старого підходу:**
- ❌ Всі клієнти використовують один інтерфейс
- ❌ Складніше управління VTEP
- ❌ При видаленні потрібно шукати конкретний VTEP за IP

### Bridge мережа
- **Мережа**: 172.16.0.0/16
- **Gateway**: 172.16.0.1
- **DHCP pool**: 172.16.0.2 - 172.16.255.254
- **Призначення**: L2 мережа для клієнтів через VXLAN

## 🔐 Безпека клієнтів

Кожен клієнт отримує повну конфігурацію безпеки:

### Firewall правила
- INPUT: Дозволено тільки з VPN та локальної мережі
- SSH/Winbox: Доступ тільки через WireGuard
- WAN: Заборонено всі вхідні з'єднання крім WireGuard

### Мережева ізоляція
- ether1 = WAN (інтернет)
- ether2-5 = Bridge (локальна мережа)
- WiFi вимкнено за замовчуванням

### Аутентифікація
- SSH тільки по ключам
- Пароль генерується випадково для кожного клієнта

## 🔄 Процес додавання клієнта

### Автоматична генерація імені та VNI

Імена клієнтів та VNI генеруються автоматично з IP адреси:

**Формат імені:** `R6{last_octet}`

**Формат VNI:** `6{last_octet}` (конкатенація)

**Приклади:**
- IP: `10.66.66.2` → Ім'я: `R62`, VNI: `62`
- IP: `10.66.66.12` → Ім'я: `R612`, VNI: `612`
- IP: `10.66.66.100` → Ім'я: `R6100`, VNI: `6100`

### Кроки додавання

1. **Генерація ключів**
   ```bash
   # Ім'я генерується автоматично
   make add-client IP=10.66.66.12
   ```
   - Створюється приватний ключ (X25519)
   - Генерується публічний ключ
   - Створюється preshared ключ
   - Зберігається в `client_keys/R612.json`

2. **Додавання на сервер**
   - Додається WireGuard peer `R612`
   - Створюється VXLAN інтерфейс `vxlan-R612` з VNI `612`
   - VXLAN додається до bridge
   - Генерується конфігурація клієнта

3. **Імпорт на клієнта**
   ```bash
   # Конфігурація: generated_configs/R612.rsc
   /import file=R612.rsc
   ```

## 🔄 Процес видалення клієнта

```bash
make remove-client IP=10.66.66.12
# Автоматично визначить ім'я: R612
```

Виконується:
1. Видалення WireGuard peer
2. Видалення VXLAN інтерфейсу (bridge port видаляється автоматично)

## 📊 Діагностика

### На сервері
```routeros
# Перевірка WireGuard peers
/interface wireguard peers print

# Перевірка VXLAN інтерфейсів
/interface vxlan print

# Перевірка bridge портів
/interface bridge port print where bridge=br_x

# Перевірка DHCP leases
/ip dhcp-server lease print
```

### На клієнті
```routeros
# Перевірка WireGuard з'єднання
/interface wireguard peers print

# Перевірка VXLAN
/interface vxlan print

# Перевірка отриманого IP
/ip dhcp-client print
```
