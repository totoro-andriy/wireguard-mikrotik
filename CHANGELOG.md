# Changelog

Всі значущі зміни в цьому проекті будуть документовані в цьому файлі.

Формат базується на [Keep a Changelog](https://keepachangelog.com/uk/1.0.0/),
і цей проект дотримується [Semantic Versioning](https://semver.org/lang/uk/).

## [1.1.0] - 2026-01-30

### ✨ Додано
- **Повністю автоматичне додавання клієнтів** - тепер просто `make add-client`
  - Автоматичний пошук першого вільного IP у діапазоні 10.66.66.2-254
  - Автоматична генерація криптографічних ключів
- **Автоматична генерація імен клієнтів** з IP адреси у форматі `R6{last_octet}`
- **Автоматичне обчислення VNI** з IP адреси: `6{last_octet}` (конкатенація)
  - IP 10.66.66.5 → VNI 65
  - IP 10.66.66.12 → VNI 612
  - IP 10.66.66.100 → VNI 6100
- Можливість вказати конкретний IP: `make add-client IP=10.66.66.12`

### 🔄 Змінено
- **ВАЖЛИВА ЗМІНА АРХІТЕКТУРИ VXLAN**: Замість використання VXLAN vteps, тепер створюється окремий VXLAN інтерфейс для кожного клієнта
- Кожен VXLAN інтерфейс має унікальну назву `vxlan-<client_name>` та унікальний VNI
- Всі VXLAN інтерфейси додаються до одного bridge на сервері
- Оновлено шаблон конфігурації клієнта для використання VXLAN з `remote-address` замість vteps
- Оновлено tasks для додавання/видалення peers
- Makefile команди: `make add-client IP=10.66.66.12` (без NAME параметра)
- Playbooks автоматично генерують client_name з IP адреси
- Оновлено документацію для відображення нової логіки

### 🐛 Виправлено
- Видалено застарілий параметр `parseable` з конфігурації ansible-lint

## [1.0.0] - 2026-01-25

### ✨ Додано
- Повна структура Ansible проекту для управління WireGuard на MikroTik
- Роль `wireguard_mikrotik` для управління WireGuard peers
- Playbooks для додавання/видалення клієнтів
- Playbook для перегляду конфігурації сервера
- Playbook для отримання публічного ключа сервера
- Скрипт генерації WireGuard ключів (`generate_keys.py`)
- Скрипт конвертації приватного ключа в публічний (`get_public_key.py`)
- Скрипт масового додавання клієнтів з CSV (`bulk_add_clients.py`)
- Автоматичний скрипт налаштування проекту (`setup.sh`)
- Скрипт валідації Ansible файлів (`validate.sh`)
- Автоматичні тести для перевірки генерації ключів (`test_keys.py`)
- Makefile з швидкими командами
- Шаблони конфігурацій для MikroTik клієнтів (.rsc)
- Шаблони конфігурацій для Linux/Windows клієнтів (.conf)
- Повна документація (README.md, QUICKSTART.md, PROJECT_SUMMARY.md)
- Приклад CSV файлу для масового додавання
- Приклад Ansible Vault для безпечного зберігання паролів
- Конфігурація для ansible-lint та yamllint
- .gitignore для захисту чутливих даних
